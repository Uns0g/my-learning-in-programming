Estrutura Do HTML

Para construir qualquer página você deve estruturá-la com as tags. A estruturação é muito simples de entender e ainda mais simples de executar

Principais Tags

As tags simplesmente classificam um elemento da página. A palavra tag significa etiqueta, uma etiqueta real marca um produto com alguma coisa, a tag faz o mesmo.

Se você ver uma tag <p> por exemplo, ela indica que aquela parte da página é um parágrafo, se ver uma tag <strong> indica que aquela parte do site está em negrito e que será pronunciada em tom mais grave por leitores de página, se ver uma tag <em> indica que aquela parte será marcada em itálico e terá ênfase na pronúncia por leitores de página e assim se sucede para cada tag. O que está inserido entre o <> indica o que uma tag "marca" na página.

Tabelas

Um conjunto de tags feitas específicamente para construir tabelas.

As tabelas que você vê pela Web são constituídas de cinco tags, a principal que declara onde há uma tabela é a <table>, ela se segue de outras duas a <thead> e a <tbody> que formam a "cabeça" e o "corpo" da tabela respectivamente, na <thead> vão os títulos e na <tbody> vai todo o conteúdo mesmo da tabela

Na <thead> vão as tags <th>, as de cabeçalho da tabela (Table Header), elas são usadas normalmente para títulos, mas não são exclusivamente para isso

Na <tbody> vão as tags <td>, as de dados da tabela (Table Data), elas são usadas para conter qualquer tipo de dado.

Toda essa lista de tags que você está cendo na verdade é uma tabela com várias <tr> com 3 <td> cada.
Listas As listas abarcam seis tags, três de "declaração" de listas e três de itens da lista. As de declaração são <ul>, <ol>, <dl> que são respectivamente: lista não ordenada (Unordered List), lista ordenada (Ordered List) e lista de definição (Definition List) e as de itens são <li>, <dt>, <dd>, respectivamente: item de lista (List Item), termo de definição (Definition Term) e descrição (Definition Description)

Tags De Formatação

Essa não é nenhuma classe de tags específicas, mas sim tags que eu gosto de chamar por esse nome, porque o que elas fazem é precisamente isso, formatam texto

As tags <b> e <strong> deixam o texto em negrito, a diferença é que a <strong> é mais usada por questões de acessibilidade e a tag b caiu em desuso

As tags <i> e <em> deixam o texto em itálico a diferença é a mesma da anterior... questões de acessibilidade

A tag <small> deixa o texto pequeno e tem o mesmo efeito que atribuir uma font-size de 0.8em para um elemento <span>. Não há nenhuma proibição de usar essa tag, eu não uso, mas se quiser usar para não ter que setar2 um valor no CSS sinta-se livre

As tags <sup> e <sub> deixam o texto superecrito assim e subescrito assim respectivamente

As tags <q> e <blockquote> mostram uma citação, a diferença é que o <q> define uma citação pequena e coloca o texto entre aspas, como se fosse uma pequena citação, uma frase, um slogan ou algo assim, enquanto que o <blockquote> define uma longa citação, talvez até um discurso e não leva aspas.

Uma tag muito pouco usada hoje em dia é a <u> que define o texto como underline (sublinhado), assim, mas essa tag caiu em desuso devido ao fato de que raramente se usa sublinhado nos textos hoje, até mesmo em links não se vê muitos, e também por conta do atributo text-decoration que quando recebe valor de underline deixa o texto sublinhado tambéml

As tags <s> e <del> passam uma linha sobre o texto, como que riscando-o, a diferença é que o <s> só indica que aquela sentença no texto era verdadeira, mas agora está desatualizada, é muito usada para indicar preços antigos, mostrar que algo está em promoção. Enquanto isso o <del> mostra um texto que realmente poderia ser desconsiderado, deletado

A Estrutura Propriamente Dita

A estrutura se dá pela hierarquia dos elementos, os elementos "pai" contém os elementos "filhos", por isso você vê muitos programadores usando as classes container e content para se referir a determinados elementos. Os containers (recipientes ou só containers mesmo) guardam os contents (conteúdos) dentro deles e ditam algumas regras.

Novamente vou propor que pense que as páginas que você vê pela web são como caixas na tela, porque elas realmente são, essa analogia faz com que você entenda melhor Você que complica autor! o funcionamento da estrutura é simples meu Deus. Ora o que for feito dentro de uma caixa vai afetar as caixas que estão dentro dela.

Nesse exemplo você pode ver que a <div> card tem 3 outros elementos "filhos" que são três outras divs dentro dela (podia ser qualquer outro elemento). as divs que estão dentro dela (filhas) não extrapolam os limites da div principal que a contém (pai), por conta disso forma-se um card (cartão) que nada mais é que um retângulo na tela contendo normalmente um título, uma imagem, talvez uma descrição e um "rodapé" (parte de baixo) que vai algum texto ou botão.

É importante compreender qual elemento contém qual para saber realmente estruturar as páginas e também para extilizar depois, já que todo o estilo cascata do CSS é a seleção de elementos da hierarquia do HMTL

Montando uma página

O HTML é um documento de texto como qualquer um, mas o que transforma um documento em html é a extensão do arquivo que deve ser .html e as tags no texto propriamente dito.

IDE

Para montar uma página html (que é um documento) você pode abrir um editor de texto e já começar a escrever, depois é só salvar esse documento como html, porém NÃO FAÇA ISSO! É muito mais aconselhável que você use uma IDE que é um Ambiente de Desenvolvimento Integrado (do inglês Integrated Development Environment) que, por sua vez, é um programa que te auxilia na criação de outros programas desktop, aplicações web, sites, aplicações mobile etc.

A IDE que eu uso é o Visual Studio Code, mas você pode usar qualquer outra que preferir, pode até usar uma online como o replit.

A IDE que eu vou usar para explicar tudo por aqui (eu acho) é o VSCode. Não irei explicar sobre a instalação e os primeiros passos com uma IDE como o Visual Studio Code, porque no site do VSCode já tem um guia preparado para te ensinar a usar o VSCode, então, se você nunca usou uma IDE é bom começar por lá, ou pelo manual de instalação e uso da sua IDE de preferência.

Execução

Para começar no Visual Studio Code você deve primeiramente criar uma pasta em algum lugar do seu computador e acessar ela clicando em File/OpenFolder e escolher a pasta indo ao caminho onde a criou.

Para criar um arquivo clique sobre o ícone de novo arquivo e automaticamente será pedido um nome para o arquivo, coloque index.html que é o nome dado ao principal documento de uma página, o índice.
